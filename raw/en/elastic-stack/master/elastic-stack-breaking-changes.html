Breaking changesedit

Before you upgrade, you must review the breaking changes for each product you use and make the necessary changes so your code is compatible with 8.4.0.

  • APM 8.4.0 breaking changes
  • Beats 8.4.0 breaking changes
  • Elasticsearch 8.4.0 breaking changes
  • Elastic Security 8.4.0 breaking changes
  • Kibana 8.4.0 breaking changes
  • Logstash 8.4.0 breaking changes
  • Make sure you check the breaking changes for each minor release up to 8.4.0.
  • If you are using machine learning datafeeds that contain discontinued search or query domain specific language (DSL), the upgrade will fail. In 5.6.5 and later, the Upgrade Assistant provides information about which datafeeds need to be updated.
